def is_valid(url):
    try:
        parsed = urlparse(url)
        if parsed.scheme not in set(["http", "https"]):
            return False
        return not re.match(
            r".*\.(css|js|bmp|gif|jpe?g|ico"
            + r"|png|tiff?|mid|mp2|mp3|mp4"
            + r"|wav|avi|mov|mpeg|ram|m4v|mkv|ogg|ogv|pdf"
            + r"|ps|eps|tex|ppt|pptx|doc|docx|xls|xlsx|names"
            + r"|data|dat|exe|bz2|tar|msi|bin|7z|psd|dmg|iso"
            + r"|epub|dll|cnf|tgz|sha1"
            + r"|thmx|mso|arff|rtf|jar|csv"
            + r"|rm|smil|wmv|swf|wma|zip|rar|gz)$", parsed.path.lower())

    except TypeError:
        print ("TypeError for ", parsed)

extract_next_links(url, resp) gives you a URL and the response from the server (i.e. the page on that URL). You are supposed to take the response and process it in order to extract the hyperlinks from it, and return them in a list of strings. The response has the following fields available:
resp.url: the actual url of the page
resp.status: the status code returned by the server. 200 is OK, you got the page. Other numbers mean that there was some kind of problem.
resp.error: when status is not 200, you can check the error here, if needed.
resp.raw_response: this is where the page actually is. More specifically, the raw_response has two parts:
resp.raw_response.url: the url, again
resp.raw_response.content: the content of the page!
is_valid(url) gives you a URL and you are supposed to decide whether to crawl it or not. If you decide to crawl it, return True; otherwise return False. The function already has a few things for which it returns False. You will need to add a few more, as you encounter collections of pages that may look like traps.
Running/Restarting the crawler
If you need to stop the crawler, press Ctrnl-c. If you need to restart it from where it left off, simply run it again. If you need to start a fresh crawl from scratch, first delete the file frontier.shelve, and then run the crawler again.

Inspecting the Log
Your crawler's log file is under Logs/Worker.log. Make sure that you inspect that Log file in order to understand what happened during the test crawls.

Server Cache status codes
These are all the cache server error codes:

600: Request Malformed

601: Download Exception {error}

602: Spacetime Server Failure

603: Scheme has to be either http or https

604: Domain must be within spec

605: Not an appropriate file extension

606: Exception in parsing url

607: Content too big. {resp.headers['content-length']}

608: Denied by domain robot rules

You may ignore some of them, but not all.
